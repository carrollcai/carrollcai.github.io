Ios、Andriod连Chrome调试

2019-02-25

前言

最近沉迷于移动端调试无法自拔。之前移动端调试，一般通过线上发版或者是Charles代理抓包。

主要是参考这篇文章。

通过chrome浏览器调试手机页面(IOS和Android)

实操Android

下面的教程是本人已经run的，所以本教程还是很可靠的。我使用的是小米手机（小米play）作为测试机，电脑为Mac，chrome版本为70+。

1.在安卓设备上开始开发者模式。MIUI开启开发者模式以及使用chrome调试移动设备上打开的网页

实际步骤，设置 -> 我的设备 -> 全部参数 -> 连续点击MIUI版本，最终显示“您已处于开发者模式”，开发者模式已经开通。

再开启USB debugger，设置 -> 更多设置 -> 开发者选项 -> USB调试。

2.将设备通过USB连上PC，在chrome浏览器里输入，chrome://inspect/#devices

3.当手机打开一个网页时，chrome点击对应的页面，点击inspect，理论上会白屏，或者是http not found, 404.

4.翻墙。

访问chrome-devtools-frontend.appspot.com，如果访问不通，就说明chrome的调试功能被墙了。

设置host或者翻墙（可以通过shadowsocks，买一个会员，或者向身边朋友借账号）。

添加hosts

5.最终页面显示出来，可支持移动端和pc同步操作。

实操Ios

safari调试

可以直接连手机，直接通过safari调试。

chrome调试

Ios的稍微复杂一点，需要安装几个插件。

根据这个github上的过程安装。https://github.com/RemoteDebug/remotedebug-ios-webkit-adapter。homebrew请自行搜索安装。

也同样需要设置hosts。

设置完，连上手机，应该就能看到chrome inspect出现手机浏览网页的记录。

如果能加载DOM元素，但是就是无法渲染页面，一直白屏。这个问题暂时没有解决方案。
